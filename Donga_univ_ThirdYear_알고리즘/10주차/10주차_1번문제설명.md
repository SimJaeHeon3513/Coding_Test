# Tree Construction

## 문제 설명

\<그림 1\>과 같은 이진 트리 T를 고려하자. 그림에서 트리의 단말노드는 대문자로, 내부노드는 소문자로 나타나 있다.  
이런 트리를 컴퓨터에서 표시하는 방법 중 하나는 다음과 같다.

어떤 tree의 두 subtree의 모양을 `( left_tree ) ( right_tree )`로 표시하고, 전체 트리는 `( ( left_tree ) ( right_tree ) )`로 표시된다.  
즉 하나의 루트와 두 개의 단말노드 A B 를 가진 트리는 `( A B )` 로 표시된다.

따라서 트리 T의 경우 입력모양은

```txt
((A B)((C D)E))
```

이 된다.

단 단말노드를 나타내는 문자와 괄호를 위한 문자 `'('`, `')'` 사이에는 하나의 공백(space) 문자가 있다.

입력파일에 트리정보가 주어지면 여러분은 이를 읽은 후, 대응하는 트리를 만들어야 한다.  
트리의 내부노드의 이름을 붙이는 규칙은 다음과 같다.

- 루트는 `'x'` 로 이름한다.
- 루트 이하의 트리를 preorder 순회하면서 만나는 순서대로 번호 `(i ≥ 0)`를 증가시키면서 이름을 붙인다.

그림에서 보인 T에 위의 규칙을 적용하면

```txt
x=0, y=1, z=2, t=3
```

이 된다.

주어진 입력에 대응하는 트리를 구성한 후, 그 트리를 `preorder` 순회, `inorder` 순회한 결과를 아래 예처럼 차례대로 보여라.

---

## 입력

입력의 첫 번째 줄에는 테스트의 개수를 나타내는 `K`가 주어진다.

이어서 `K`개의 테스트에 관한 입력이 주어진다.  
각 테스트는 한 줄에 주어진다.

- 트리단말에 표시될 문자의 길이는 최대 8자
- 한 줄에 있는 문자의 수는 최대 50,000개 (공백 포함)

---

## 출력

각 테스트 케이스에 대하여,

1. 첫 번째 줄에는 입력에서 읽은 문자를 그대로 출력한다.
2. 이어 `"Preorder"`를 출력한 후 preorder traverse 한 결과를 출력한다.
3. 이어 `"Inorder"`를 출력한 후 inorder traverse 한 결과를 출력한다.

입력값을 그대로 보일 때, 각 문자열 또는 괄호 문자는 하나의 공백으로 구분된다.

---

## 예시 트리

```txt
          x
        /   \
       y     z
      / \   / \
     A   B t   E
          / \
         C   D
```

---

## 예시 입력

```txt
1
((A B)((C D)E))
```

---

## 예시 출력

```txt
((A B)((C D)E))

Preorder
x y A B z t C D E

Inorder
A y B x C t D z E
```
