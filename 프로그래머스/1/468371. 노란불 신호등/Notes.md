# 문제 분석 및 접근 방식
- 주기가 있는 것을 보고 주기 시간과 최소공배수를 이용해야겠다고 생각했음
- math의 LCM 혹은 gcd를 활용하여 직접 lcm을 구현
- 주기는 초록불과 노란불의 합
- 노란불 판단 방법은 bool 타입의 변수를 사용

# 풀이과정
1. 프로그래머스에서 math를 사용불가하여 gcd를 활용하여 lcm을 직접 구현
2. 모든 신호드의 주기를 1차원 배열에 저장
3. 첫 번째 주기와 두 번째 주기의 LCM을 구하고, 그 LCM과 세 번째 주기와 LCM을 구하고 총 N 번째의 LCM을 구함
4. 전체 신호등의 LCM의 시간(t)동안 bool 타입의 all_yellow를 True로 선언하고 G <= 현재 위치 < G + Y의 조건을 만족하지 않는다면 최소 하나가 노란불이 겹치지 않으므로 False를 반환하고 종료함
5. 만약 LCM까지의 루프를 모두 도는데에 성공하고 all_yellow가 True라면 현재 t를 반환하고 종료
6. 하지만 모두가 노란불이 되지 않는다면 -1을 반환하고 종료

# 시행착오 및 배운점
- 로직을 구현하는거 자체를 생각해내지 못했음
- 이런 주기를 가지는 루프에서 현재 위치를 나타낼 때 (t-1) % 주기 즉, 나머지 연산을 이용하면 됨

# 개선가능한 포인트
- 주기와 LCM과 같은 핵심적인 포인트는 알아챌 수 있으나 이걸 가지고 로직을 짜내는 연습을 하면 문제를 해결하는데에 도움이 될 것임
